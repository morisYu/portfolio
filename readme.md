# 기획안  

## 항목  

1. 웹사이트의 제목  

    > 쿠팡(쿠폰이 팡팡 터진다)와 같이 제목에 웹사이트에 정체성을 표현  
      최종 오픈 전까지 제목은 언제든지 바뀔 수 있음  
      도메인과 함께 고려헤야함.

2. 웹사이트의 도메인  

    > 웹사이트의 제목과 같으면 좋고, 웹사이트 제목과 관련이 있어야 함.  

3. 웹사이트의 목적

    > 사용자에게 어떤 유익을 줄 수 있는가?  
      사람들이 원할만한 것을 작성  
      뚜렷한 목적이 있어야 함.  

    - 웹사이트의 역할과 목적  
      + 정보 전달 미디어  
      + PR과 광고 미디어  
      + 엔터테인먼트 콘텐츠  
      + 온라인 학습 시스템  
      + 온라인 커뮤니티  
      + 유통시스템  
      + 가치 플랫폼    

4. 목표 고객  

    > 타킷층을 구분해야함  
      user segmentaition: 비슷한 성격을 가진 타깃 집달을 구분하는 것.  
      웹사이트가 가장 핵심적으로 삼을 타깃층을 결정.  
      1차, 2차, 3차 순으로 타깃층을 선정  

5. 웹사이트의 서비스 기획  

    > 사용자 시나리오: 사용자들의 웹사이트에서 행동 패턴을 작성.  
      사용자 시나리오를 토대로 웹사이트가 갖추어야 하는 콘텐츠와 기능을 도출  
      타깃이 여럿 있는 경우 타깃별로 사용자 시나리오 필요  
    
6. 메뉴 기획  
   
    > 1차메뉴: 세부 서비스 항목들을 그룹화한 대표 제목  
      2차메뉴: 1차 메뉴의 구분에 알맞은 세부적 서비스들(1차 메뉴의 하위메뉴)  

7. 웹사이트 콘텐츠 구성  

    > 웹사이트 콘텐츠는 사용자에게 데이터베이스 형태로 제공되는 텍스트 정보를 포함하며 특정 사이트에 있는 모든 구성 요소들.  
      목적과 서비스가 구체적이고 단순할수록 기획이 큰 성과를 낼 수 있음  
      ✨ 웹 기획 업무 중 제일 먼저 진행되는 가장 중요한 부분  

8. 차별성  

    > 다양한 소비자들의 기호와 니즈에 맞추어야 함.  
      다른 유사 웹사이트와의 차별성이 있어야 함.  

    - 차별성이란 구체적이다.  
        + 모양과 내용
        + 형태와 퀄리티  
        + 유통 채널  
        + 가격  

9. 벤치마킹  

    > 특정 분야에서 우수한 상대를 표적으로 삼아 자기 기업과의 성과 차이를 비교하고, 극복하기 위해 그들의 뛰어난 운영 프로세스를 배우면서 혁신을 추구하는 경영기법.  
      

    - 좋은 웹사이트가 공통적으로 갖추고 있는 가장 기본적인 내용  
        + 웹사이트의 유용성: 쓸모있는 가치  
        + 양질의 콘텐츠: 지속적인 업데이트  
        + 사용자 편의성: 사용자 경험과 사용자 인터페이스  
        + 사이트 신뢰성: 기술과 보안 서비스  
        + 브랜드 디자인: 시각적 아름다움과 브랜드 이미지  
        + 홍보와 재미: 재방문을 유도  
        + 사용자 참여 유도: 상호호혜적 관계  
        + 개인화 서비스: 개별 고객의 취향과 필요
